So... what IS AI? 🤔

You know how when you first picked up a bass guitar, you had to learn slowly — which fret, which string, which finger. But now you've played so much that your fingers just know where to go without thinking?

AI learned the same way — except it didn't practice guitar. It read millions and millions of books, websites, and conversations. So many that it learned how humans think, write, and solve problems. Now when you ask it something, it doesn't have to think from scratch — it already knows patterns from everything it read.

📚

What it learned

Books, science papers, code, stories, conversations — billions of pages. It learned how ideas connect and how to explain things clearly.

🤔

What it can NOT do

It can't actually slap a bass string. It only lives inside computers. And — this is important — it forgets you every time you close the chat! Like amnesia. We'll fix that later.

The Amnesia Problem: Every new chat, the AI wakes up knowing nothing about you. Imagine if your guitar teacher forgot everything about you at the start of every lesson — your skill level, your favourite songs, what you practised last week. You'd have to explain it all from scratch! The whole point of Projects and Skills is to fix this forgetting problem.

Chapter 2 — 🏏 Agents are Like Fielding Positions!
Sub-agents are like
fielding positions! 🏏
🔄

Switching from guitars to cricket for a second!

We've been talking about bands and music. Now we're going to explain a tricky AI concept using cricket instead — because cricket actually explains it perfectly. Don't worry, we'll get back to guitars soon. Just imagine you've walked off stage and onto the pitch! 🏏

You know how a slip fielder has ONE job (catch edges), the wicketkeeper has ONE job (catches behind), and a mid-on has ONE job (stop straight drives)? If you asked your slip fielder to also keep wickets AND bat AND coach, they'd do everything badly!

AI agents work the same way. When a task is complicated, the main AI (the coach) breaks it into small jobs and gives each job to a specialist agent. Each one does their job perfectly. Then all the results come back together — just like a brilliant fielding setup that takes a catch!

🔍

Agent 1:
The Opener

Goes in first. Looks at the incoming information and figures out what's there. Reports back: "here's what we've got."

⚖️

Agent 2:
The Wicketkeeper

Misses nothing. Checks for errors, missing bits, anything that doesn't look right. Very detail-focused!

🏃

Agent 3:
The Pace Bowler

The specialist with serious skill. Takes the data from the other agents and applies the clever techniques — calculates, analyses, uses deep knowledge to find the answer.

📝

Agent 4:
The Commentator

Takes all the results from the other agents and turns it into a clear, readable final report for you!

🎯

The Orchestrator:
Head Coach

Doesn't field — directs. Decides who does what, in what order. This is Claude — the LLM brain!

Why not just ONE agent for everything? Imagine asking your slip fielder to run in from the boundary AND bat AND keep wickets all at the same time! They'd get confused and mess up. Narrow, focused jobs = much better results. The coach (Claude) just says "you do THIS, you do THAT" — and the whole team clicks.

Chapter 3 — 🏏 Tokens are Like Overs in a T20!
Tokens are like
overs in a T20 match! 🏏

Back to cricket for this one — because it's the perfect way to understand something called TOKENS.

The AI doesn't read whole words. It reads in tiny chunks called tokens. A token is roughly half a word. "Cricket" = 2 tokens. "Hit that six!" = 4 tokens. Every conversation has a limit on how many tokens can fit — just like a T20 match has exactly 20 overs and no more. Once the overs run out, the innings is over!

You have to be smart about how you use your overs. Use too many early overs on the wrong thing, and you have no overs left when you really need them. An AI conversation works exactly the same way.

⚠️

What if ALL 20 overs are used up?

In a T20, once all 20 overs are bowled, the innings ends — no more batting! When the token limit fills up, the AI starts forgetting the beginning of the conversation to make room for new messages. It's like if the scorecard ran out of space and they had to erase the early overs to write new ones! Keep your conversations focused on one topic and you'll never run out of overs.

🎸 3 Rules — Bass Player Edition!

🎯

One Song Per Set

Don't mix topics in one chat. Just like you don't play jazz in the middle of a rock gig! One topic = one conversation.

📝

Write Down the Riffs

Save important decisions to a file after each chat. Next time, load the summary — not the whole conversation. Way fewer overs wasted!

🚫

Don't Paste Whole Songs

Never paste a huge document in. It fills up the whole setlist with one song! Put it in a reference file instead — Claude reads just what it needs.

Chapter 4 — 💻 Now Let's Talk About Coding!
Coding is like
writing a bass tab! 🎸

You know how a bass tab tells your fingers exactly what to do — which string, which fret, in what order? Code is exactly the same thing, but for a computer. Instead of telling your fingers what to do, you're telling the computer exactly what to do, step by step.

🎸

Bass Tab vs Code — Same Idea!

Bass Tab: G|---5---3---5---7---|
This tells your fingers: play fret 5, then 3, then 5, then 7, on the G string.

Code: print("Hello World!")
This tells the computer: display the words "Hello World!" on screen. Exact. Precise. In order.

〰️〰️〰️

Your First Python Program!

The most popular coding language right now is called Python. Why Python? Because it reads almost like English sentences! Let's look at real code:

my_first_program.py
# This is a comment — the computer ignores it! # Comments are just notes for YOU (like writing in your tab book) # Tell the computer your name name = "Aryan" # Tell it your favourite sport sport = "cricket" # Tell it your bass guitar type bass = "electric bass" # Print a message using all three! print("Hi! My name is", name) print("I love", sport, "and playing", bass) # OUTPUT (what you see on screen): # Hi! My name is Aryan # I love cricket and playing electric bass

What are "variables"? In the code above, name, sport, and bass are variables. A variable is like a box that holds information. You label the box (name it) and put something inside. In guitar terms — it's like a preset on your bass amp. You save your settings (high gain, heavy bass) and give it a name. Next time, just call the preset!

〰️〰️〰️

Making Decisions in Code — The If Statement

The most powerful thing in coding is making decisions. "IF this happens, do THAT. Otherwise, do something else." Computers are brilliant at following these rules exactly.

cricket_decision.py
# The cricket situation runs_needed = 12 balls_left = 6 # Make a decision based on the situation if runs_needed <= balls_left: print("Play safe! We have time.") elif runs_needed <= balls_left * 2: print("Hit boundaries! Need 2 runs per ball.") else: print("Go for SIXES! Nothing to lose!") # With runs_needed=12, balls_left=6: # Hit boundaries! Need 2 runs per ball.
🎸

Guitar Version of an If Statement!

IF the crowd is quiet → play it gentle and soft
ELIF the crowd is warming up → add some energy
ELSE (crowd is going wild!) → full distortion, maximum volume 🔥

Your brain makes these decisions every gig. Code just writes them down so a computer can do the same!

〰️〰️〰️

Loops — Playing the Same Riff Repeatedly!

When you practise a bass riff, you play it again and again until it's perfect. In coding, when you want to do something many times, you use a loop. It's literally the computer repeating the same action — just like you repeating that tricky fret pattern!

bass_practice.py
# Practice this riff 5 times for practice_round in range(5): print("Round", practice_round + 1, ": G|---5---3---5---7---|") # Round 1 : G|---5---3---5---7---| # Round 2 : G|---5---3---5---7---| # Round 3 : G|---5---3---5---7---| # Round 4 : G|---5---3---5---7---| # Round 5 : G|---5---3---5---7---| # Instead of writing it 5 times manually, # the loop does it for you!

This is why coders are lazy (in a good way!): A good coder never writes the same thing twice. They write it once, put it in a loop or a function, and let the computer repeat it. In bass terms — you don't re-tune your bass for each song. You tune it once and play all night!

〰️〰️〰️

Functions — Your Saved Bass Presets!

Your bass amp probably has preset settings you can save — clean sound, slap bass, distortion. You set them up once and then just punch the button whenever you need that sound. In coding, a function is exactly that — you write some code once and give it a name, then call that name whenever you need it!

cricket_scorer.py
# Define a function (like saving a preset) def calculate_run_rate(runs, overs): run_rate = runs / overs return run_rate # Now USE the function (punch the preset button!) team_a = calculate_run_rate(180, 20) # 180 runs in 20 overs team_b = calculate_run_rate(156, 20) # 156 runs in 20 overs print("Team A run rate:", team_a) # → 9.0 print("Team B run rate:", team_b) # → 7.8 print("Team A wins!")
🏆

Why functions are brilliant!

Without a function, you'd have to write the run-rate formula every single time. With a function, you write it ONCE and call it as many times as you want — for any team, any match, any tournament! Write once. Use forever. That's engineering thinking. 🎯

Chapter 5 — 🔧 Build Your Own AI Assistant!
Put it all together —
build your own AI! 🚀

Now you understand AI AND coding. Here's the amazing thing: you can build your own AI assistant using code! It won't be as big as Claude, but it will be yours, it will work, and you'll understand every single part of it.

Let's imagine you build a "Cricket Brain" — an AI assistant that knows everything about cricket, helps you track match scores, and gives you batting advice. Here's how you'd build it step by step:

1
📋 Write the Skill (The Training Manual)

In plain English, write what your Cricket Brain knows. "You are a cricket expert. You know all the rules of cricket. When someone asks for batting advice, always consider the pitch conditions first." This is your system prompt — the recipe book!

2
💻 Write the Code (The Agent Loop)

In Python, write the loop: ask Claude your question → Claude reads the skill → Claude uses it → Claude answers. This is literally just the code we looked at, connected to the Claude API (an API is like a telephone line to Claude).

3
🗄️ Add a Memory File

Save important things to a text file after each conversation. Next time, load the file first. Now your Cricket Brain remembers your batting average, your strongest shots, your practice history — even after you restart the computer!

4
🔌 Connect to Real Data (The MCP Bit)

The most advanced step! Write code that lets your Cricket Brain read from a live database — real match scores, real statistics. Now when you ask "how many runs did I score this season?" it actually looks them up instead of guessing!

5
🎸 Make It Yours

Change the skill for your bass guitar! "You are a bass guitar teacher. You know music theory. You give practice advice based on what the student is working on." Same code. Different skill. Your own Guitar Brain!

What the Code Actually Looks Like

cricket_brain.py ← this is YOUR file!
# Your Cricket Brain — a real AI assistant you built! # Step 1: Import the tools you need (like plugging in your bass amp) import anthropic # Step 2: Write your skill (the recipe book) my_skill = """ You are a cricket expert and bass guitar coach. You know everything about cricket and music. Always give specific, practical advice. Never make up statistics — if you don't know, say so. """ # Step 3: The agent loop — ask question, get answer def ask_cricket_brain(question): client = anthropic.Anthropic() response = client.messages.create( model = "claude-sonnet-4-6", system = my_skill, messages = [{"role": "user", "content": question}] ) return response.content[0].text # Step 4: Use it! answer = ask_cricket_brain( "How do I improve my cover drive technique?" ) print(answer) # That's it! You just built an AI assistant in ~20 lines of code!

This is REAL code. If you installed Python on your computer, got an API key from Anthropic, and ran this file — it would actually work. You would have built a real AI assistant. The code above is genuinely what professional AI developers write — just with more details added for bigger projects. You already understand the concepts. The code just writes them down in a language computers understand.
